In the interest of transparency, this page discloses how AI has been used in the development of ngx-formbar.

The core principles and architecture of this library were conceived and designed without AI involvement. The original concept was inspired by [How to Make Forms in Angular REUSABLE](https://www.youtube.com/watch?v=o74WSoJxGPI) by [DecodedFrontend](https://www.youtube.com/@DecodedFrontend) and developed from there. AI tools were introduced later as an implementation aid.

All AI-generated work was human-directed and reviewed. Design decisions, architecture, and the overall direction of the project are driven by the maintainers. AI served as an implementation tool under human supervision.

## Areas Where AI Was Used

In the following areas AI played a major role, besides helping out with tedious refactorings or smaller helper functions.

### Expression Engine

The expression engine in `@ngx-formbar/core` was fully implemented with AI assistance. This includes the expression parser, evaluator, safe method allowlists, and operator handling.

### Schematics

The code generators in `@ngx-formbar/schematics` and the shared AST utilities in `@ngx-formbar/setup` were fully implemented with AI assistance. This includes the control, group, block, and register generators, as well as the TypeScript AST manipulation and file handling utilities.

### Testing

The unit test suites across the project were fully implemented with AI assistance.

### Documentation

AI assisted in writing, restructuring, and reviewing documentation pages, including this documentation site.
