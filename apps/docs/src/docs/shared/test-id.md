A base test ID is always generated for each control.

### Default Behavior

By default, the test ID is equal to the control's name (the key used in the configuration), prepended with its parent's test ID if it's part of a group. For example, a control with name `'firstName'` inside a group with test ID `'user-details'` would have a final test ID of `'user-details-firstName'`. If the control is at the root level, its test ID is `'firstName'`.

### Customizing Test IDs

Customize how test IDs are generated by providing a global `testIdBuilderFn` when you set up Formbar using `provideFormbar`. This function receives the control's content, its name, and its parent's test ID (if any) and should return the test ID string. It is used for all controls, groups, and blocks.

```typescript name="app.config.ts"
provideFormbar({
  // ...
  globalConfig: {
    testIdBuilderFn: (content, name, parentTestId) => {
      return `${parentTestId ? parentTestId + '_' : ''}${content.type}-${name}`;
    },
  },
});
```

If no `testIdBuilderFn` is provided, the default behavior described above is used.
