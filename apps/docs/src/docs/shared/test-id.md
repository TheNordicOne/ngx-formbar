To make testing easier, a base test id will always be generated for you.

### Default Behavior

By default, the test id is equal to the control's name (the key used in the configuration), prepended with its parent's test id if it's part of a group. For example, a control with name `'firstName'` inside a group with test id `'user-details'` would have a final test id of `'user-details-firstName'`. If the control is at the root level, its test id would simply be `'firstName'`.

### Customizing Test IDs

You can customize how test IDs are generated by providing a global `testIdBuilderFn` when you set up Formbar using `provideFormbar`. This function receives the control's content, its name, and its parent's test ID (if any) and should return the desired test ID string. It is used for all controls, groups, and blocks.

```typescript name="app.config.ts"
provideFormbar({
  // ...
  globalConfig: {
    testIdBuilderFn: (content, name, parentTestId) => {
      return `${parentTestId ? parentTestId + '_' : ''}${content.type}-${name}`;
    },
  },
});
```

If no `testIdBuilderFn` is provided, the default behavior described above is used.
