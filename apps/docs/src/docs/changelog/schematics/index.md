## v2.0.0

### Overview

New package containing all code generators, previously part of `@ngx-formbar/core`. Component generators now scaffold interface-based components that align with the `@ngx-formbar/reactive-forms` v2 architecture. Generated components implement the public formbar interfaces and declare their bindings as signal `input()` fields, with no host-directive wiring or directive injection.

### Added

- **`control` generator:** Scaffolds a new control component that implements `ReactiveFormbarControl<T>` from `@ngx-formbar/reactive-forms`, declares its bindings via signal `input()`, and registers itself automatically
- **`group` generator:** Scaffolds a new group component that implements `ReactiveFormbarGroup<T>`, uses `<ngxfb-control-outlet />` to project child controls, and registers itself automatically
- **`block` generator:** Scaffolds a new block component that implements `FormbarBlock<T>` and registers itself automatically
- **`register` generator:** Discovers all existing components in the project and bulk-registers them as `ComponentRegistrationEntry` values

### Changed

- Generated control, group, and block components now implement the corresponding `@ngx-formbar/reactive-forms` interface and expose their inputs as signals, replacing the previous host-directive-plus-directive-injection pattern
- Group templates render children through `<ngxfb-control-outlet />` instead of relying on a host-directive-projected outlet
- Generated registrations use `ComponentRegistrationEntry` from `@ngx-formbar/core` (`staticComponent()` / `loadComponent()`), supporting both static and lazy component registrations

### Removed

- **`--hostDirectiveHelperPath` CLI option:** Removed from the `control`, `group`, and `block` generators. Generated components no longer apply host directives, so the helper path is obsolete. The `--viewProviderHelperPath` option still applies to `control` and `group` generators.
- Generated components no longer reference `hostDirectives` or `inject(NgxfbControlDirective)` / equivalent group/block directives

### Migration

No migration steps required for new generations. Install as a dev dependency to replace the generators previously in `@ngx-formbar/core`:

```shell
npm install -D @ngx-formbar/schematics
```

Update any scripts using `@ngx-formbar/core:<generator>` to `@ngx-formbar/schematics:<generator>`. If your build pipeline passed `--hostDirectiveHelperPath`, drop the flag; it is no longer accepted. Existing components generated by the v1 schematic continue to work; follow the `@ngx-formbar/reactive-forms` v2.0.0 migration guide to convert them to the interface-based pattern.
